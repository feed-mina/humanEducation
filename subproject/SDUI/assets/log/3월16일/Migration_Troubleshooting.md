# 마이그레이션 오류 해결 보고서 (2026-03-16)

로컬 테스트 중 발생한 `bootRun` 실패 및 Flyway 마이그레이션 오류에 대한 원인 분석과 조치 내용을 기록합니다.

## ❌ 발생한 오류 (Error Details)
- **증상**: `bootRun` 실행 시 Flyway가 `V34` 마이그레이션 수행 중 실패하며 서버 구동 중지.
- **오류 메시지**: `오류: "system_prompt_template" 칼럼은 "ui_metadata" 릴레이션(relation)에 없음`
- **원인**: 새롭게 구현된 AI 일본어/인터뷰 기능에서 페르소나별 시스템 프롬프트를 DB 레벨에서 주입하기 위해 해당 컬럼 사용을 시도했으나, 실제 테이블 스키마에 컬럼이 존재하지 않아 SQL 실행이 거부됨.

## 🛠 조치 내용 (Resolution Steps)

### 1. 데이터베이스 스키마 수정
- **V34 마이그레이션 추가**: `V34__add_system_prompt_column.sql` 파일을 생성하여 `ui_metadata` 테이블에 `system_prompt_template (TEXT)` 컬럼을 추가하도록 구성했습니다.
- **마이그레이션 순서 조정**: 기존에 `V34`였던 일본어 채팅 페이지 등록 스크립트를 `V35`로 변경하여, 컬럼이 생성된 직후에 데이터가 삽입되도록 순서를 맞췄습니다.

### 2. 백엔드 코드 동기화 (Entity & DTO)
- **UiMetadata 엔티티 수정**: `UiMetadata.java` 엔티티 클래스에 `systemPromptTemplate` 필드를 추가하고 JPA 매핑(@Column)을 완료했습니다.
- **UiResponseDto 수정**: 프론트엔드로 데이터를 전달하는 DTO에 해당 필드를 추가하고, 엔티티에서 데이터를 복사하는 생성자 로직을 업데이트했습니다.

## ✅ 결과 확인
- 마이그레이션 파일들이 `V34`(컬럼 추가) -> `V35`(데이터 추가) 순서로 정렬되어, Flyway가 순차적으로 스키마를 업데이트할 수 있는 상태가 되었습니다.
- **추천 조치**: `bootRun`을 재실행하여 마이그레이션이 성공하는지 확인해 주세요.
