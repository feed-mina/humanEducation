## Q&A 보고서: 자전거 대여량 예측(GRU) 오류 분석

### Q1. 학습 데이터 분리 단계

**문제점(Cause):** `train_test_split` 함수를 사용하여 데이터를 랜덤으로 분리하면 시계열 데이터의 시간 순서가 깨집니다. 시계열 예측에서는 과거 데이터를 통해 미래를 예측해야 하는데, 랜덤 분리로 인해 미래 데이터가 학습 데이터에 포함될 수 있습니다.

**부작용(Effect):** 데이터 누출(data leakage)이 발생하여 모델이 과적합되고, 실제 미래 예측 성능이 떨어집니다. 모델이 미래 정보를 미리 학습하게 되어 현실적인 평가가 불가능합니다.

### Q2. GRU 입력 데이터 형태 변환 (Reshape)

**문제점(Cause):** 데이터를 `(샘플수, 컬럼수, 1)`로 reshape하면 GRU의 시간 축(time step)에 피처(feature)가 들어가게 됩니다. GRU는 첫 번째 차원을 시퀀스 길이로 해석하는데, 여기서는 각 피처를 개별 시간 스텝으로 오해합니다.

**부작용(Effect):** 모델이 시계열 패턴을 학습할 수 없고, 피처 간의 관계를 시간적 순서로 잘못 해석하여 예측 정확도가 매우 낮아집니다.

### Q3. 테스트 데이터 예측 시점

**문제점(Cause):** 테스트 데이터만 가지고 시퀀스를 생성하면, 시퀀스 길이(예: 24시간)만큼의 앞부분 데이터가 예측할 수 없습니다. 예를 들어, 20일 00시를 예측하려면 19일의 데이터를 필요로 하는데, 테스트 데이터만으로는 불가능합니다.

**부작용(Effect):** 테스트 데이터의 일부 행이 예측되지 않거나, 예측 결과가 불완전해집니다.

**해결책:** 훈련 데이터의 마지막 부분을 테스트 데이터와 연결하여 시퀀스를 생성해야 합니다. 예를 들어, 훈련 데이터의 마지막 24시간 + 테스트 데이터를 사용하여 예측합니다.

### Q4. 예측 결과의 도메인 적합성

**문제점(Cause):** 회귀 모델은 수식에 따라 음수 값을 예측할 수 있는데, 자전거 대여량은 0대 이상이어야 합니다. 음수 예측을 그대로 제출하면 비현실적입니다.

**부작용(Effect):** 제출 결과가 도메인 규칙을 위반하여 평가에서 불이익을 받거나, 실제 적용 시 오류가 발생합니다.

**필요한 후처리 코드:**

```python
y_pred = np.maximum(0, y_pred)  # 음수 값을 0으로 클리핑
```
